# TodoAgent 产品功能说明书

## 1. 文档信息
- 产品名称：TodoAgent
- 文档类型：产品功能说明书（Functional Specification）
- 适用范围：Web 端（公开站点 + 登录后工作台）
- 版本：v1.0（基于当前仓库实现）

## 2. 产品概述
TodoAgent 是一个以「任务执行 + AI 助手」为核心的生产力产品。用户可在多项目工作区中管理待办任务，通过 AI 对话与内联续写提升执行效率，并在任务完成后自动沉淀分析与 SOP，形成可复用的个人工作方法库。

核心价值：
- 让任务管理从“记录”升级到“执行与复盘”。
- 通过 AI 降低写作、拆解、整理与检索成本。
- 通过专注统计与任务分析增强长期行为反馈。

## 3. 目标用户与使用场景
目标用户：
- 个体创作者、自由职业者、学生、知识工作者。
- 有持续任务管理、内容整理、AI 协作需求的用户。

典型场景：
- 每日任务计划与优先级管理。
- 在任务正文中进行快速编辑、结构化记录与图片粘贴。
- 与 AI 结合当前任务上下文进行问答、总结、生成。
- 完成任务后自动产出可替代性分析和 SOP 增量内容。

## 4. 系统范围与运行环境
技术与依赖：
- 前端：React + TypeScript + Vite。
- 路由：React Router。
- 认证：Clerk（含未配置时 fallback）。
- 存储：localStorage（本地）+ Supabase（云同步）。
- AI：Moonshot/Kimi（支持代理接口模式）。
- 计费：Stripe 订阅 + 服务端权益判断。

运行要求：
- Node.js >= 20。
- 本地开发命令：`npm install`、`npm run dev`。
- 构建命令：`npm run build`，预览命令：`npm run preview`。

## 5. 功能架构总览
产品功能分为 8 大模块：
- 公共站点（首页、博客、方案页、隐私与条款）。
- 登录认证与路由保护。
- Todo 工作台（多项目 + 任务全生命周期管理）。
- AI 对话与工具调用。
- 统计分析与 SOP 沉淀。
- Agent Home 与文档中心。
- 专注力模块（番茄钟/种树）。
- 计费与配额控制。

## 6. 功能详细说明

### 6.1 公共站点模块
功能点：
- 首页展示产品价值、CTA 入口、FAQ。
- 博客列表与详情页浏览。
- Solutions Hub 及方案详情展示。
- 隐私政策与服务条款页面（多语言内容）。

输入/输出：
- 输入：匿名用户访问公开 URL。
- 输出：营销内容展示与跳转至登录/工作台入口。

### 6.2 登录认证与访问控制
功能点：
- 登录用户可访问 `/app/*` 工作台路由。
- 未登录用户访问工作台时触发登录流程。
- 当 Clerk 公钥缺失时，应用仍可降级渲染（fallback）。

输入/输出：
- 输入：用户访问受保护页面。
- 输出：鉴权通过后进入工作台；否则引导登录。

### 6.3 Todo 工作台模块
功能点：
- 项目管理：新增项目、切换项目、重命名项目。
- 任务管理：新增、编辑、删除、完成/取消完成。
- 任务属性：优先级（P0/P1/P2）、截止时间、计时器信息。
- 视图能力：按优先级筛选（ALL/P0/P1/P2）。
- 排序规则：未完成优先，其次按优先级与创建时间。
- 编辑增强：Markdown 列表行为、图片粘贴/缩放/下载、选区提问。
- 效率能力：全局撤销/重做、内联 AI 续写（末尾触发）。

输入/输出：
- 输入：用户在任务列表中的创建/编辑/筛选操作。
- 输出：任务列表与任务详情状态实时更新，并触发持久化。

### 6.4 AI 对话模块（ChatPanel）
功能点：
- 支持全局会话与任务级会话。
- 会话历史管理与自动命名。
- 上下文注入：Todo 内容、文档引用、图片附件。
- 可选工具调用：网页搜索、URL 抓取、Skill 导入（依赖配置）。

输入/输出：
- 输入：用户消息、附件、工具开关状态。
- 输出：AI 回复、会话记录更新、必要时工具调用结果回填。

约束：
- 图片附件最多 4 张，单张大小不超过 5MB。

### 6.5 统计分析与 SOP 模块
功能点：
- 当任务被标记完成后，自动触发分析流程。
- 生成“可替代性分析卡片”用于复盘与能力评估。
- 基于完成任务增量更新 SOP 文档，形成方法沉淀。

输入/输出：
- 输入：任务完成事件。
- 输出：分析结果写入任务维度数据；SOP 文档增量更新。

### 6.6 Agent Home 与文档中心
功能点：
- 文档管理：上传、预览、编辑、删除。
- 支持文档类型：`.md`、`.markdown`、`.txt`、`.docx`。
- Skill 文档管理：供 AI 对话工具使用。
- Agent 个性配置（3 种人格选项）及 AI 总结刷新。

输入/输出：
- 输入：用户上传和维护文档、配置 Agent 风格。
- 输出：文档库与 Agent 配置状态更新，并可被对话模块消费。

### 6.7 专注力模块（Focus）
功能点：
- 25 分钟专注计时。
- 种树反馈机制。
- 今日与累计专注统计展示。

输入/输出：
- 输入：用户开始/结束专注。
- 输出：专注记录与统计数据更新。

### 6.8 计费与配额模块
功能点：
- 免费用户每日 Agent 调用次数限制（UTC 口径）。
- 配额耗尽时展示付费墙弹窗。
- 订阅支付：Stripe Checkout 跳转与回调确认。
- 服务端权益判定后控制前端可用能力。

输入/输出：
- 输入：用户发起 Agent 调用或订阅行为。
- 输出：放行调用或触发付费引导；订阅状态回写权益。

## 7. 核心业务流程

### 7.1 任务创建与编辑流程
1. 用户创建任务。
2. 系统写入默认任务结构（未完成、默认优先级）。
3. 用户编辑正文，系统同步提取标题并更新编辑历史栈。
4. 状态变化触发本地持久化，随后进入云同步队列。

### 7.2 任务完成 -> 分析 -> SOP 沉淀流程
1. 用户将任务标记为完成。
2. 系统更新完成计数并提示反馈。
3. 系统触发 AI 分析，生成任务级分析卡片。
4. 系统将该任务经验增量并入 SOP 文档。

### 7.3 项目切换与数据隔离流程
1. 用户切换项目。
2. 系统按项目 ID 加载对应 Todo、会话、文档与统计数据。
3. 项目间数据隔离，互不覆盖。

### 7.4 本地持久化与云同步流程
1. 业务状态变化后写入 localStorage。
2. 同步层以防抖策略触发上行推送至 Supabase。
3. 初始化或需要时执行云端拉取并回填本地状态。

## 8. 数据对象说明（产品视角）
- Project：项目实体，承载任务与会话上下文。
- Todo：任务实体，包含内容、优先级、完成状态、时间信息。
- Conversation：对话会话实体，包含消息序列与会话元数据。
- Analysis：任务分析实体，关联 Todo 并保存可替代性结果。
- SOP：流程沉淀文档，按任务完成持续更新。
- WorkspaceDoc：用户文档实体，支持上传/编辑/删除。
- FocusMap：专注统计实体（今日/累计等）。

## 9. 边界与约束
功能约束：
- 文档上传暂不支持 `.doc`。
- 内联 AI 续写仅在编辑器末尾触发，且返回短文本（2-5 字）。
- 联网工具依赖 Supabase Edge Function 及相关外部 Key 配置。

安全与访问约束：
- 工作台路由受认证控制。
- URL 抓取存在内网地址拦截等安全限制。

运营约束：
- 免费额度为每日 10 次 Agent 调用（UTC）。

## 10. 非功能要求
性能与体验：
- 云同步采用防抖写入，减少频繁网络请求。
- 对话输入区与布局支持动态交互（如宽度调整、输入区自适应）。

可靠性：
- 本地持久化与云端回填并存，降低数据丢失风险。
- 分析流程具备失败兜底与重试控制机制。

安全性：
- 服务端进行鉴权与 webhook 验签。
- 部署层包含 CSP 与抓取安全约束。

可维护性与扩展性：
- 模块按 components/hooks/utils/api/pages 分层。
- 对话工具调用与 Edge 能力可扩展。

## 11. 验收标准（建议）
- 任务全生命周期（增删改查、完成、筛选、排序）可用且状态正确。
- 多项目数据隔离可验证，切换不串数据。
- 对话模块可正常收发消息并保留历史会话。
- 任务完成后可生成分析结果并更新 SOP。
- 文档中心支持上传、编辑、删除与对话引用。
- 免费额度与付费墙策略按日生效，订阅后权益可即时感知。
- 本地刷新后数据可恢复，云同步开启时跨端可回填。

## 12. 后续演进建议（可选）
- 增加自动化测试（单元/集成/E2E）与回归基线。
- 增加更细粒度权限模型（团队/协作场景）。
- 丰富统计维度（周期趋势、任务耗时分布、执行质量评分）。
- 提供 SOP 版本管理与对比能力。
